SquidgyCMS uses a combination of HTML and a custom markup which is similar to that used on wikis. A detailed HTML reference will not be provided here, but w3schools provide bery good tutorials and references.

SquidgyML

The SquidgyCMS Markup Language is the method by which site users can use pre-defined Blocks in their pages.

Simple blocks are inserted by using the following syntax: [[Block::moduleID-blockTitle]]

More complex blocks can require that you provide some information, this is specified as follows: [[Block::moduleID-blockTitle::dataString]]

Where:
moduleID
This is the ID of the module (note that this may be diferent from its name.
blockTitle
This is the title of the block that you wish to insert.
dataString
This can be either a single piece of information or a more complex data structure. In the latter case the following syntax is used: {key1:value1|key2:value2} Any number of key/value pairs are allowed, though in the case that only one is given the pair separator can be removed: {key:value} The keys to use will be specified by the module, as will likely values.
